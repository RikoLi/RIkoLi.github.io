倒数第二个暑假来了，之前我做了什么呢？

关于某门课

最想吐槽的第一个说。是的，就是这学年的人工智能课程，这门课让我体验了各种意想不到的奇妙操作，final project体感极差。这与授课老师和课程本身内容无关，令人迷惑的是它的final project安排。

首先，在最开始，在这门课刚开课的时候我还对这门课程充满了热情，提前很早构思了期末大作业的内容、实现方式、时间安排，仓库也搭起来了，辅助工具也写好了不少，一切看起来似乎完美按照计划进行。

当我首次在课上听说hw提供协助资源的时候，我还是很高兴的，这比起当初那门缺少各种训练资源的机器学习final可是好过不少的了。

然后喜闻乐见的事情果然发生了：时间推进到期中，我们得知final prj要在能够在hw指定的某开发板上部署，更绝的是这块开发板只支持caffe下的一些算子，而且只能做CV相关的模型部署。也就是说，我们要在不足半个学期的时间内，在还有其他几门硬核专业课的情况下，从零开始入门这块开发板的使用，最可怕的是，如果我们真的按照这个要求做下去，我们之前构思准备过的内容即将全部木大白费，这谁顶得住啊.jpg

这操作实在是太迷惑了，我甚至觉得整个期中之后遇到的最迷惑行为非这个“限定开发/部署环境”的所谓“大作业”莫属。一门算法理论课的大作业，重点本应当放在学生对内容的理解而产生的idea以及实现idea的过程上，而不是什么限定框架、限定平台这种愚蠢的事情。此规定一出，大半个班级哗然，谁都知道这意味着自己做过的努力很可能付之东流，大家不断向老师提问如果期末prj不用这块板子会如何，老师看起来很无奈，也表示过对于一门课程而言这样做确实很奇怪，但是似乎也无能为力。

这么看来，问题似乎很清楚了，这板子估计是hw新产品，性能有待检测，需要测试工作。而这门课搞不好只是个悲惨的牺牲品，我们就是D级人员，天然的测试用户。后续的种种迹象让我不得不坚信这种可能性。毕竟，如果是一门普通的算法类课程大作业，谁会说出“请大家记录开发板使用过程中的bug，在最后展示中反馈出来，我们会视情况进行加分”这种荒唐的话？起初我怀疑是不是只有我们学院被拿来祭天了，找yxd问了一通计院的情况，好了，明白了，全军覆没，一个都跑不了。

时间紧迫，我和yxd商量了一下跨学院合作的可能性，发现这是个好方法，最后的展示不同学院是分开的，也就是说我们完全可以share一些公共资源，于是和yxd，zsb一起开始搞。接下来就是噩梦般的几天，配置开发板的环境，从物理连接到后面编译配置再到最后模型导入等等过程中碰到了一堆匪夷所思的问题，前面说了这设备极可能是新产品，所以出了问题除了官方基本找不到solution，更绝的是，设备相关文档所在位置让人一言难尽，找起来简直要命…遇到了太多麻烦，实在不想回忆，那几天每次搞到后半夜，然后第二天早上起来接着搞，要知道这可是考试周前不到一个星期，其他课程还处于基本没怎么复习的状态，实在是太弱智了。

终于在某日凌晨2:30左右，这板子可算是能运行起来了，通知了通宵写BS的zsb，总算松了口气。yxd人在国外，速成了一份看起来非常牛逼的报告，一通狂吹我们的模型产生了多么fancy的效果，我和zsb凑了几组数据贴到PPT里，在计院那边展示开始一小时前总算是基本成型了一份作品。

配环境期间我调查了一下同学们的进度，结果不出所料：

  • xxx组板子环境没配出来，还在配环境
  • xxx组环境配置不出来
  • xxx组完全不知怎么办，很绝望…
  • …

只有一组的状态是能跑起来分类模型，太真实了吧，这简直就是全村的希望.jpg

一个小插曲：临近考试周前几天，估计是课程群里抱怨声引起了注意，hw赶制了一个算是demo的视频（反正俺没看）说是肯定能成功部署在板子上，结合最后展示时候的情况来看，估计这视频做的应该就是个SSD目标检测的demo，因为展示的时候简直是SSD大会，很多组做的全是SSD目标检测，而且基本就没做什么修改，就直接展示了一下最基本的SSD功能。

接下来，最搞笑的事情发生了，这可能是整个课程final prj中最精彩、最高潮的环节，作为亲历人，得知这个消息的时候当场笑喷了。zsb参加计院展示期间，一直给我发同学们的作品，看了一下全是非常高大上的选题，什么DRL，什么金融量化分析都出来了，我们觉得和人家一比简直就是乡下人进城，直到zsb给我发了这条消息：

（聊天记录节选）

zsb：我们拿了1000块钱。。。
zsb：吃饭又着落了。。。
zsb：*有
riko：woc
riko：stm
zsb：但是不好的事情在于。。
zsb：要交给hw。。（指我们的模型）
…
zsb：我们说我们根据我们调整过的vgg训练了ssd。。。
riko：捂脸流泪.jpg

您好，请问该用什么表情.jpg

是这样，hw承诺在展示后由他们的工程师进行打分评选出几组优秀作品，其中一等奖1组，二等奖2组，三等奖3组，获奖组将得到价值xxxx元的奖励。

我们的确就是拿信电这边助教给的baseline模型随便调了调结构，本来打算着做个分类器说一下这板子能运行就完事儿了，zsb可能是展示鬼才，把这东西和SSD结合了一下，然后就被选中了…虽然不知道中了几等奖，但是好歹有回报，z…zsb展示牛逼！（破音）焼肉行くぜ，平海路准备启动！

计院那边展示告一段落，现在只剩下我们自己学院了。好在有另一门课做了个performance还不错的模型，直接拿过来wrap正合适。这套系统从头到尾虽然都是自己实现的，但是部署平台完全用不到那块开发板，只好丢了个分类器到板子上说用这个板子测试了模型效果，以此强行加入开发板的使用环节。左搞右搞总算把展示用的PPT编排好了，还是在通信原理考试前一天晚上做的，这宝贵的复习时间啊…（后面通信原理考的是什么垃圾）

7.4早上硬着头皮上去开着全英文的PPT用中文吹了一遍我们的prj（这种放英文讲中文的行为被一个朋友称为耍流氓x），结束之后还被hw官方安利了一波什么什么比赛…果然，这课程的大作业的目的确实变味了，哈哈.jpg

下午跑去唱anisong，四点骑车回yq参加颁奖仪式。结果又是出乎意料，中了个二等奖（再 生 産），奖品是组员一人一个500R的移动打印机，原来不是和计院一样是奖金啊…Top 1是tjk他们的抽烟检测，当之无愧，当时配环境的时候他们在旁边，简单问了下，做的挺不错的，主要是板子用的很充分，很合乎hw那边的期待。（我们不是一个班的，但是没想到最后展示是一起展示，原来还想冲一下top的，笑）

总之，这门课是一门真实的期末大戏，不枉碎碎念了这么多。结果令人哭笑不得，也难以评价，因为最后它已经走得太远了。

望贵世一大不要再搞这种弱智的事情。

暑假

难得的，一个没有被安排的明明白白的暑假。虽说有了实习，但是做的是兴趣相关的领域，没觉得是个负担，反倒很期待，希望开始之后也能继续保持热情吧。

大三终于解决了语言智商税的问题，这个暑假又没有在学校上的小学期，或许能抽出点时间约人出来玩玩，难得。

整理一下，大概有这么些事：

  1. lab留下的一些小事
  2. 实习
  3. 回顾一些东西，读书
  4. 外语日常温习
  5. 锻炼！

接下来就是大四了，总算迎来了久违的空闲，可以抛开乱七八糟的事情专心投进lab了，希望一切顺利。加把劲骑士！

7.26

  • 模型终于收敛了，有了baseline结果，提升空间还很大

7.27

很开心的一天，晚上和很久不见的初中同学一起出去吃了顿饭，聊了很多东西，吹了很多水，很高兴看见大家生活得都有滋有味。尤其和csb聊到他开始学画画，看了看他的作品，画得有模有样，不愧是专门上了插画课，想想自己似乎已经很久没动过笔了，有些惭愧，感觉是时候捡起来再系统学学了。

聊完发现自己假期这几天来除了去实习也没干什么比较有意义的事情，还是太摸了，之前定的事情还需要细化。

首先告诉自己的应该是，不要浮躁。感觉这些日子实习有点急于求成，但是这种实验性质的工作本身不确定性就很强，有时候出问题再怎么着急也没用，还是应该思考思考再思考，梳理整个流程，考虑哪些地方可能出问题，出哪些问题，用什么方法检验问题，用什么途径解决问题，着急只能让脑子变得更不好用。

之前本来想安排假期复习一遍传统machine learning算法，顺便coding实现一遍，无奈那本《机器学习实战》放寝室没带回来，只好等开学之后安排吧，反正大四倒是也好安排这些。另外可能需要提前考虑一下毕设的问题了，最好在开学之后一个月以内就确定下来，早点开始总是好的。哦对，11月再冲次GRE试试，希望还能再高几分吧。

言归正传，具体安排一下假期每天做什么：

  • 上午实习工作
  • 下午除了工作，抽时间背GRE单词，背2个list，复习前一天2个list
  • 晚上看看视频学画画，看看动画什么的当休息了
  • 周末写实习周总结
